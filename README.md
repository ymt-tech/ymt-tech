## Development Practices

個人開発でもチーム開発を意識した運用を心がけています。

- ブランチ戦略: `main` / `develop` / `feature/*` / `fix/*` / `docs/*`
- コミット: [Conventional Commits](https://www.conventionalcommits.org/)（例: `feat:`, `fix:`, `docs:`）
- レビュー: Issue → feature ブランチ → Pull Request → マージ
